A tabela EKKO¶

Para nossos propsitos, usaremos as seguintes colunas da tabela EKKO:

  • EBELN - Contm o nmero do pedido de compra.

  • ERNAM - Contm o funcionrio do grupo de compras que autorizou o pedido.

  • LIFNR - Contm o identificador exclusivo do fornecedor que recebeu o pedido.

  • KNUMV - Contm o link para o conjunto de condies associadas ao pedido de compra.

A tabela EKPO¶

Para nossos propsitos, usaremos as seguintes colunas da tabela EKPO:

  • EBELN - Contm o nmero do pedido de compra.

  • EBELP - contm o identificador do item de linha.

  • TXZ01 - Contm uma descrio textual do material.

  • MATNR - Contm o identificador exclusivo do material.

  • MENGE - Contm a quantidade de material solicitado.

  • NETPR - Contm o preo lquido efetivo do material pedido.

  • NETWR - Contm o valor lquido efetivo do material pedido (ou seja,MENGE * NETPR).

A tabela KONV¶

Para nossos propsitos, usaremos as seguintes colunas da tabela KONV:

  • KNUMV - Contm o identificador exclusivo para o conjunto de condies.

  • KPOSN - Contm o identificador do item de linha.

  • KSCHL - Contm o tipo de uma condio.

  • KAWRT - Contm o valor base que o sistema usa para calcular o preo da condio.

  • KBETR - Contm o preo efetivo.

Os seguintes tipos de condio so relevantes para este estudo de caso:

  • PBXX - Preo bruto

  • RB00 - Descontos absolutos

  • NAVM - dedues fiscais

  • SKTO - descontos vista relacionados s metas de pagamento

  • WOTB - Preo efetivo

A varivel a seguir contm todos os tipos de condies relevantes.
